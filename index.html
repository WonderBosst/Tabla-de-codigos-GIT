Instalación y configuración
Git Git es un sistema de control de revisión distribuido, escalable y rápido con un conjunto de comandos inusualmente rico que proporciona operaciones de alto nivel y acceso completo a los elementos internos. -global, -system, -local, -f, -e, -z, git init, git init [nombre del proyecto], git clone nombredeusuario@host:/path/to/repository, git add, git commit –m “El mensaje que acompaña al commit va aquí”, git config --local user.email tuemail@ejemplo.com, git status, git push origin, etc.
config Puede consultar/establecer/reemplazar/deshabilitar opciones con este comando. -global, -system, -local, -f, -e, -z, git config [<file-option>] [--type=<type>] --add <name> <value></value>
help Sin opciones y sin o proporcionados, la sinopsis del comando git y una lista de los comandos Git más utilizados se imprimen en la salida estándar. git help
bugreport Captura información sobre la máquina del usuario, el cliente de Git y el estado del repositorio, así como un formulario que solicita información sobre el comportamiento que observó el usuario, en un solo archivo de texto que el usuario puede compartir, por ejemplo, con la lista de correo de Git, para para reportar un error observado. -o, -s git help [-a|--all] [--[no-]verbose] [--[no-]external-commands] [--[no-]aliases], git help [[-i|--info] [-m|--man] [-w|--web]] [<command>|<doc>], git help [-g|--guides], git help [-c|--config], git help [--user-interfaces], git help [--developer-interfaces]
Obtener y Crear Proyectos
init Crea un repositorio Git vacío o reinicializa uno existente -q, -b $ cd /path/to/my/codebase, $ git init (1), $ git add . (2), $ git commit (3)
clone Clona un repositorio en un nuevo directorio -v,-q,-n,-l,-s,-j,-o,-b,-u,-c,-4,-6 $ git clone git://git.kernel.org/pub/scm/.../linux.git my-linux, $ cd my-linux, $ make
Instantáneas básicas
add Este comando actualiza el índice utilizando el contenido actual que se encuentra en el árbol de trabajo, para preparar el contenido preparado para la próxima confirmación. -n,-v,-i,-p,-e,-f,-u,-N,-A $ git add Documentation/\*.txt, $ git add git-*.sh
status Muestra el estado del árbol de trabajo. Muestra las rutas que tienen diferencias entre el archivo de índice y la confirmación HEAD actual, las rutas que tienen diferencias entre el árbol de trabajo y el archivo de índice, y las rutas en el árbol de trabajo que no son rastreadas por Git (y no son ignoradas por gitignore). -v,-s,-b,-z,-u git status
diff Muestra los cambios entre confirmaciones, confirmación y árbol de trabajo, etc. Muestra los cambios entre el árbol de trabajo y el índice o un árbol, los cambios entre el índice y un árbol, los cambios entre dos árboles, los cambios resultantes de una combinación, los cambios entre dos objetos blob o los cambios entre dos archivos en el disco. -p,-s,-u,-U,-w,-z,-M,-D,-C,-b,-I,-a,-R,-S,-O git diff
commit Registrar cambios en el repositorio. Cree una nueva confirmación que contenga el contenido actual del índice y el mensaje de registro dado que describa los cambios. -a,-i,-p,-o,-n,-z,-u $ edit hello.c, $ git rm goodbye.c, $ git add hello.c, $ git commit
notes Agregar o inspeccionar notas de objetos. Agrega, elimina o lee notas adjuntas a objetos, sin tocar los objetos mismos. -f,-F,-q,-n,-v,-c,-C git notes [list [<object>]], git notes add [-f] [--allow-empty] [-F <file> | -m <msg> | (-c | -C) <object>] [<object>], git notes copy [-f] ( --stdin | <from-object> [<to-object>] ), git notes append [--allow-empty] [-F <file> | -m <msg> | (-c | -C) <object>] [<object>]
restore Restaura archivos de árbol de trabajo. Restaura las rutas especificadas en el árbol de trabajo con algunos contenidos de una fuente de restauración. Si se rastrea una ruta pero no existe en la fuente de restauración, se eliminará para que coincida con la fuente. -S,-W,-q,-m,-2,-3,-p,-s $ git switch master, $ git restore --source master~2 Makefile, $ rm -f hello.c, $ git restore hello.c
reset Restablece el HEAD actual al estado especificado. -q,-z,-p,-N git reset
rm Elimina archivos del árbol de trabajo y del índice. Elimina los archivos que coincidan con pathspec del índice o del árbol de trabajo y el índice. -n,-q,-f,-r
mv Mueve o cambia el nombre de un archivo, un directorio o un enlace simbólico. -v,-n,-f,-k git mv [>opciones>] >fuente>…​ >destino>>/destino>
Ramificación y fusión
branch Lista, crea o elimina ramas -a,-d,-D,-m,-M,-c,-C,-l,-f,-i,-v,-q,-t,-u,-r git branch -d -r origin/todo origin/html origin/maN, $ git branch -D test
checkout Cambia ramas o restaura archivos de árbol de trabajo. Actualiza archivos en el árbol de trabajo para que coincidan con la versión en el índice o el árbol especificado. -b,-B,-l,-q,-m,-d,-t,-f,-2,-3,-p git checkout [-q] [-f] [-m] [>rama>], git checkout [-q] [-f] [-m] --detach [>rama>], git checkout [-q] [-f ] [-m] [--detach] >commit>
switch Cambia ramas. Cambiar a una sucursal específica. El árbol de trabajo y el índice se actualizan para que coincidan con la rama. Todas las confirmaciones nuevas se agregarán a la punta de esta rama. -c,-q,-m,-d,-t,-f git switch [>opciones>] [--no-guess] >branch>, git switch [>opciones>] --detach [>start-point>], git switch [>opciones>] (-c|-C) >nuevo -branch> [>punto de inicio>], git switch [>opciones>] --orphan >nueva-rama>
merge Une dos o más historias de desarrollo. Incorpora cambios de las confirmaciones nombradas (desde el momento en que sus historias divergieron de la rama actual) en la rama actual. -n,-e,-s,-X,-m,-F,-v,-q,-S git merge (--continuar | --abortar | --salir)
mergetool Ejecuta herramientas de resolución de conflictos de fusión para resolver conflictos de fusión. -y,-q,-O git mergetool [--tool=>herramienta>] [-y | --[no-]prompt] [>archivo>…​]
log Mostrar registros de confirmación. -q,-L git log [>opciones>] [>rango de revisión>] [[--] >ruta>…​]
stash Guarda los cambios en un directorio de trabajo sucio. -u,-q,-p,-u,-a,-l,-m git stash list [>opciones de registro>]
tag Crea, enumere, elimine o verifique un objeto de etiqueta firmado con GPG. -a,-m,-F,-e,-s,-u,-f,-i git tag -v [--format=>formato>] >nombre de etiqueta>…
worktree Administra múltiples árboles de trabajo. -n,-f,-v, git worktree add [-f] [--detach] [--checkout] [--lock [--reason >string>]] [-b >nueva-rama>] >ruta> [>commit-ish>]
Compartir y actualizar proyectos
fetch Descarga objetos y referencias de otro repositorio. -V,-b,-n,-f,-z,-v git fetch [>opciones>] [>repositorio> [>refspec>…​]]
pull Obtenga e integre con otro repositorio o una sucursal local. -a,-f,-t,-p,-j,-k,-o,-4,-6 git pull [>opciones>] [>repositorio> [>refspec>…​]]
push Actualiza las referencias remotas junto con los objetos asociados. -v,-q,-d,-n,-f,-u,-o,-4,-6 git push
remote Administra un conjunto de repositorios rastreados. -v,-t,-m,-d,-p,-a git remoto [-v | --verbose]
submodule Inicializar, actualizar o inspeccionar submódulos. -f,-b git submodule [--quiet] [--cached]
Inspección y Comparación
show Muestra varios tipos de objetos. -q,-L git show [>opciones>] [>objeto>…​]
log Muestra registros de confirmación. -q,-L git log [>opciones>] [>rango de revisión>] [[--] >ruta>…​]
diff Muestra los cambios entre confirmaciones, confirmación y árbol de trabajo, etc. -p,-u,-s,-U, git diff [>opciones>] [>commit>] [--] [>ruta>…​]
difftool Muestra cambios usando herramientas diff comunes. Es un comando de Git que le permite comparar y editar archivos entre revisiones utilizando herramientas de comparación comunes. -g,-d,-y,-t,-x git difftool [>opciones>] [>commit> [>commit>]] [--] [>ruta>…​]
range-diff Compara dos rangos de confirmación (por ejemplo, dos versiones de una rama). Este comando muestra las diferencias entre dos versiones de una serie de parches, o más generalmente, dos rangos de confirmación (ignorando las confirmaciones combinadas). -p,-s,-u,-U,-W git range-diff
shortlog Resume la salida del registro de git. -s,-e,-n git shortlog
describe Dar a un objeto un nombre legible por humanos basado en una referencia disponible. -a git describe
Parchar
apply Aplica un parche a los archivos y/o al índice. -p,-N,-3,-z,-C,-R,-v,-q git apply
cherry-pick Aplicar los cambios introducidos por algunas confirmaciones existentes. -n,-e,-s,-m,-X,-S,-x git cherry-pick
rebase Vuelve a aplicar confirmaciones encima de otra sugerencia base -q,-n,-v,-C,-f,-m,-i,-x,-r,-s,-X,-S git rebase
revert Revierte algunas confirmaciones existentes. -X,-S,-n,-e,-s,-m git revert
Depuración
bisect Usa la búsqueda binaria para encontrar la confirmación que introdujo un error. git bisect >subcomando> >opciones>
blame Muestra qué revisión y autor modificaron por última vez cada línea de un archivo. -b,-L,-t,-l,-S,-P,-M,-C,-h,-c,-f,-n,-s,-e,-w git blame
grep Imprime líneas que coincidan con un patrón. -v,-E,-p,-F,-l,-z,-o,-A,-w,-f-a,-I,-i,-w,-n git grep [-a | --texto] [-I] [--textconv] [-i | --ignorar-caso] [-w | --palabra-regexp]
